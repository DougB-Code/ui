## Design Context

### Users
This UI is a live beta operator console for people managing the control plane and local harness configuration for Agent Awesome. The primary workflows are browsing operational records, verifying system health, and editing configuration safely without touching backing files directly.

### Brand Personality
Precise, calm, operator-grade. The interface should feel trustworthy, dense enough for serious admin work, and visually composed rather than flashy.

### Aesthetic Direction
Dark, high-contrast control-room styling with restrained accent color, strong information hierarchy, and compact productivity patterns. The preferred reference direction is the provided Providers mock: split-pane admin workspace, sticky controls, compact record browsing, and clear operational states.

### Design Principles
1. Separate global shell chrome from section-level workspaces and entity editing.
2. Prioritize scanability first, then editing depth second.
3. Keep operational actions visible and close to the relevant data.
4. Use dense, calm surfaces instead of oversized dashboard cards.
5. Preserve the same workspace pattern across Providers, Agents, Tools, and Workflows.
